2.12 Example: Principal Components Analysis(实例：主成分分析)

One simple machine learning algorithm, principal components analysis (PCA), can be derived using only knowledge of basic linear algebra.

主成分分析（principal components analysis，PCA）是一个简单的机器学习算法，可以通过基础的线性代数知识推导。

Suppose we have a collection of m points {x(1),⋯ ,x(m)}\lbrace x^{(1)}, \cdots, x^{(m)}\rbrace{x(1),⋯,x(m)} in Rn\R^{n}Rn and we want to apply lossy compression to these points. Lossy compression means storing the points in a way that requires less memory but may lose some precision. We want to lose as little precision as possible.

假设在 Rn\R^{n}Rn 空间中有 mmm 个点 {x(1),⋯ ,x(m)}\lbrace x^{(1)}, \cdots, x^{(m)}\rbrace{x(1),⋯,x(m)}，我们希望对这些点进行有损压缩。有损压缩表示我们使用更少的内存，但损失一些精度去存储这些点。我们希望损失的精度尽可能少。

One way we can encode these points is to represent a lower-dimensional version of them. For each point x(i)∈Rnx^{(i)}\in\R^{n}x(i)∈Rn we will ﬁnd a corresponding code vector c(i)∈Rlc^{(i)}\in\R^{l}c(i)∈Rl. If lll is smaller than nnn , storing the code points will take less memory than storing the original data. We will want to ﬁnd some encoding function that produces the code for an input, f(x)=cf(x)=cf(x)=c, and a decoding function that produces the reconstructed input given its code, x≈g(f(x))x\approx g(f (x))x≈g(f(x)).

编码这些点的一种方式是用低维表示。对于每个点 x(i)∈Rnx^{(i)}\in\R^{n}x(i)∈Rn，会有一个对应的编码向量c(i)∈Rlc^{(i)}\in\R^{l}c(i)∈Rl。如果 lll 比 nnn 小，那么我们便使用了更少的内存来存储原来的数据。我们希望找到一个编码函数，根据输入返回编码，f(x)=cf(x)=cf(x)=c；我们也希望找到一个解码函数，给定编码重构输入，x≈g(f(x))x\approx g(f (x))x≈g(f(x))。

PCA is deﬁned by our choice of the decoding function. Speciﬁcally, to make the decoder very simple, we choose to use matrix multiplication to map the code back into Rn\R^{n}Rn. Let g(c)=Dcg(c) = Dcg(c)=Dc, where D∈Rn×lD\in\R^{n×l}D∈Rn×l is the matrix deﬁning the decoding.

PCA由我们选择的解码函数而定。具体来讲，为了简化解码器，我们使用矩阵乘法将编码映射回 Rn\R^{n}Rn，即 g(c)=Dcg(c) = Dcg(c)=Dc，其中D∈Rn×lD\in\R^{n×l}D∈Rn×l 是定义解码的矩阵。

Computing the optimal code for this decoder could be a diﬃcult problem. To keep the encoding problem easy, PCA constrains the columns of DDD to be orthogonal to each other. (Note that DDD is still not technically “an orthogonal matrix” unless l=nl = nl=n.)

2.11 The Determinant(行列式)

The determinant of a square matrix, denoted det(A)\mathrm{det}(A)det(A), is a function that maps matrices to real scalars. The determinant is equal to the product of all the eigenvalues of the matrix. The absolute value of the determinant can be thought of as a measure of how much multiplication by the matrix expands or contracts space. If the determinant is 000, then space is contracted completely along at least one dimension, causing it to lose all its volume. If the determinant is 111, then the transformation preserves volume.

行列式，记作 det(A)\mathrm{det}(A)det(A)，是一个将方阵 AAA 映射到实数的函数。行列式等于矩阵特征值的乘积。行列式的绝对值可以用来衡量矩阵参与矩阵乘法后空间扩大或者缩小了多少。如果行列式是 000，那么空间至少沿着某一维完全收缩了，使其失去了所有的体积；如果行列式是 111，那么这个转换保持空间体积不变。

2.8 Singular Value Decomposition(奇异值分解)

In section 2.7, we saw how to decompose a matrix into eigenvectors and eigenvalues. The singular value decomposition(SVD) provides another way to factorize a matrix, into singular vectors and singular values. The SVD enables us to discover some of the same kind of information as the eigendecomposition reveals; however, the SVD is more generally applicable. Every real matrix has a singular value decomposition, but the same is not true of the eigenvalue decomposition.For example, if a matrix is not square, the eigendecomposition is not deﬁned, and we must use a singular value decomposition instead.

在第2.7节，我们探讨了如何将矩阵分解成特征向量和特征值。还有另一种分解矩阵的方法，称为奇异值分解（singular value decomposition，SVD），是将矩阵分解为奇异向量（singular vector）和奇异值（singular value）。通过奇异值分解，我们会得到一些与特征分解相同类型的信息。然而，奇异值分解有更广泛的应用。每个实数矩阵都有一个奇异值分解，但不一定都有特征分解。例如，非方阵的矩阵没有特征分解，这时我们只能使用奇异值分解。

2.7 Eigendecomposition(特征分解)

Many mathematical objects can be understood better by breaking them into constituent parts, or ﬁnding some properties of them that are universal, not caused by the way we choose to represent them.

许多数学对象可以通过将它们分解成多个组成部分或者找到它们的一些属性来更好地理解。这些属性是通用的，而不是由我们选择表示它们的方式所产生的。

For example, integers can be decomposed into prime factors. The way we represent the number 121212 will change depending on whether we write it in base ten or in binary, but it will always be true that 12=2×2×312 = 2\times 2\times 312=2×2×3. From this representation we can conclude useful properties, for example, that 121212 is not divisible by 555, and that any integer multiple of 121212 will be divisible by 333.

例如，整数可以分解为质因数。我们可以用十进制或二进制等不同方式表示整数 121212，但是 12=2×2×312 = 2\times 2\times 312=2×2×3 永远是对的。从这个表示中我们可以获得一些有用的信息，比如 121212 不能被 555 整除，或者 121212 的倍数可以被 333 整除。

Much as we can discover something about the true nature of an integer by decomposing it into prime factors, we can also decompose matrices in ways that show us information about their functional properties that is not obvious from the representation of the matrix as an array of elements.

正如我们可以通过分解质因数来发现整数的一些内在性质，我们也可以通过分解矩阵来发现矩阵表示成数组元素时不明显的函数性质。

One of the most widely used kinds of matrix decomposition is called eigendecomposition, in which we decompose a matrix into a set of eigenvectors and eigenvalues.

特征分解（eigendecomposition）是使用最广的矩阵分解之一，即我们将矩阵分解成一组特征向量和特征值。

2.6 Special Kinds of Matrices and Vectors(特殊类型的矩阵和向量)

Some special kinds of matrices and vectors are particularly useful.

有些特殊类型的矩阵和向量是特别有用的。

Diagonal matrices consist mostly of zeros and have nonzero entries only along the main diagonal. Formally, a matrix DDD is diagonal if and only if Di,j=0D_{i,j}= 0Di,j​=0 for all i≠ji\neq ji=j. We have already seen one example of a diagonal matrix: the identity matrix, where all the diagonal entries are 1. We write diag(v)\mathrm{diag}(v)diag(v) to denote a square diagonal matrix whose diagonal entries are given by the entries of the vector vvv . Diagonal matrices are of interest in part because multiplying by a diagonal matrix is computationally eﬃcient. To compute diag(v)x\mathrm{diag}(v)xdiag(v)x, we only need to scale each element xix_{i}xi​ by viv_{i}vi​. In other words, diag(v)x=v⊙x\mathrm{diag}(v)x=v\odot xdiag(v)x=v⊙x. Inverting a square diagonal matrix is also eﬃcient.
